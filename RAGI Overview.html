RAGI OVERVIEW

 

Ruby Asterisk Gateway Interface (RAGI) is a useful open-source framework for bridging the Ruby on Rails web application server environment and Asterisk, the open-source PBX.

 

RAGI eases the development of interactive automated telephony applications such as IVR, call routing, automated call center, and extended IP-PBX functionality by leveraging the productivity of the Ruby on Rails framework.  RAGI simplifies the process of creating rich telephony and web apps with a single common web application framework, object model and database backend.

 

RAGI is available under the BSD license.

Class Overview

 

The most important classes in RAGI to an application developer are callConnection, callHandler and callInitiate.  These three classes provide everything you need to create interactive phone applications in Asterisk using Ruby.  The other classes provide runtime support and test facilities. 

 

This section provides a high level overview of each class in RAGI.  Detailed API documentation is (or will be) located in the source files for each class.  Usage examples are provided in other sections of this document and in accompanying sample code.

 

call_connection.rb – this class provides a Ruby API for controlling functions on the Asterisk server.  Many, but not all, of the Asterisk AGI commands are implemented.  The syntax of these API is designed to be convenient and familiar to a Ruby developer and thus do not map exactly to the method name and parameterization of Asterisk AGI.

 

call_handler.rb – each application using AGI must create a subclass of CallHandler in order to implement logic for handling a call. There is one method on this object, processCall, which is called when a call is passed through RAGI from Asterisk.

 

call_initiate.rb – this class provides a convenient abstraction around AsteriskÅfs call file facility for outbound dialing.  In addition, CallInitiate specifies some conventions for configuring AsteriskÅfs extensions.conf so that calls, when answered, are handled in turn by RAGI.  In summary, RAGIÅfs callInitiate provides a mechanism to create interactive applications triggered by a dial out. 

 

Important note:  CallInitiate relies on having an Asterisk outgoing directory set up and an accompanying cron job.  The setup for this is the same as the wake up call configuration found on this page:  http://www.voip-info.org/tiki-index.php?page=Asterisk+tips+wake-up

 

call_server.rb – a server component that answers an AGI socket connection from Asterisk and spawns threads for each call connection on a particular CallHandler.  A ÅgHandlerMapÅh facility provides a means to map CallHandler implementations by class name to URIs.  These URIs are specified in the Asterisk extensions.conf to route to the appropriate handler.

 

start_ragi.rb – an example class that starts up a RAGI server with the Simon Game CallHandler.  For instructions on running RAGI within a Rails application, please see the README file.

 

config.rb – the class provides default configuration values for RAGI. In practice, developers will want to configure RAGI for their own purposes (and with their own HandlerMap) by passing in parameters to the initialize method of callServer.  In addition, defaults for things like the path to the Asterisk sound directory and wakeups are given here.  Note that you can override these for your environment by changing system environment variables.

 

test.rb – a Ruby unit test class for testing RAGI applications.

 

 

Configuration and Getting Started

 

In order to understand how to use RAGI in a real Ruby application, this section will guide you through running any one of our sample apps.  This section assumes you have Ruby installed and path and environment variables set up properly for your environment.

 

Obtain RAGI

1.  Type the following to install RAGI as a gem:   gem install ragi

 

In the directory ÅgragiÅh you will find the core class files that make up RAGI and a helpful README file.

In the directory Ågsample-appsÅh you will find a subdirectory for the simon game sample application.

 

Configure Asterisk

Please see the README file for examples on how to use RAGI in the context of a Rails application.

 

Run the Sample Application

The class Ågstart_ragi.rbÅh is used to bootstrap a server process to answer incoming RAGI connections from your Asterisk server.   This file is pre-configured to run the simon gameÅfs call handler. 

1.  cd to the director that contains start_ragi.rb

2.  type the following to execute the app:   ruby start_ragi
